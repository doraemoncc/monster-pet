# 2026-04-19 工作记录

## 小怪兽养成记 - 导航点击无反应（真正根因）

**问题**：任务中心、星币商店、家长面板点击后完全无反应  
**根因1（主要）**：`onboarding-overlay`（z-index:400）每次都挡住所有点击。原因是 `store.js` 的 `STORAGE_KEY='monster-pet-data'`，但浏览器 localStorage 里没有这个 key（可能是历史遗留旧数据或首次访问），所以 `onboardingDone` 永远是默认值 `false`，引导弹窗永远显示  
**根因2（次要）**：`initRouter()` 里 `currentPage` 初始值是 `'pet'`，调 `navigateTo('pet')` 第一行就 return，首次加载宠物页不渲染（空白）  
**修复**：
1. `store.js init()`：加旧 key 迁移（`monsterPet`/`monster_pet`/`petGame` → `monster-pet-data`）
2. `app.js initRouter()`：`currentPage = null` 强制首次触发 `page:enter`  
3. `sw.js`：版本 v1→v2，缓存策略改网络优先（顺带修复）  
**验证**：Playwright 全部通过，4页均有内容，零JS错误

## 小怪兽养成记 - Service Worker 缓存问题（已包含在上方）

**问题**：任务中心、星币商店、家长面板点击后无反应  
**根因**：Service Worker 使用"缓存优先"策略 + 固定版本号 `monster-pet-v1`，iPad 上 SW 一直服务旧缓存（含有 `STAGE_EXP` 重复声明 bug 的旧 `pet-renderer.js`），新修复的代码永远不会被加载  
**修复**：
1. `sw.js` 缓存版本升级为 `monster-pet-v2`（触发旧缓存清除）
2. 缓存策略从"缓存优先"改为"网络优先"，确保每次都加载最新代码，仅在网络失败时降级缓存  
**验证**：Playwright 测试代码逻辑正常（tasks 5399 字符、shop 4659 字符、parent 1066 字符）

**用户操作**：重新部署后，iPad Safari 强制关闭 App 重新打开即可

## 小怪兽养成记 - 页面切换后顶部空白

**问题**：点击任务中心/商店/家长面板后，页面顶部有大片空白，需要上划才能看到内容  
**根因**：`navigateTo()` 切换页面时未重置滚动位置（`window.scrollTo`），浏览器保持了上一页的 scroll offset  
**修复**：`app.js navigateTo()` 中在 `page:enter` 之前加 `window.scrollTo(0, 0)`
