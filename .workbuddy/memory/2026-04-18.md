## 小怪兽养成记 - Bug 修复（22:45）

**问题**：首次引导后底部导航栏点击无反应

**根因**：
1. **致命**：`pet-renderer.js` 第 20 行 `const STAGE_EXP = [0, 200, 600, 1500]` 与 `store.js` 第 226 行重复声明。浏览器全局作用域中 `const` 不能重复声明，导致 `SyntaxError`，整个 `pet-renderer.js` 文件不执行。虽然后续 script 标签仍会加载，但宠物页面无法渲染。
2. **次要**：`onboarding.js` 第 60 行 `store.set('pets[0].name', name)` 路径错误，`store.set` 按点号分割为 `['pets[0]', 'name']`，不会正确修改宠物名字。

**修复**：
1. 删除 `pet-renderer.js` 中的 `const STAGE_EXP` 声明（保留 store.js 中的）
2. 修改 `onboarding.js` 为直接操作数组 `pets[0].name = name` 然后 `store.set('pets', pets)`

**教训**：浏览器中多个 `<script>` 标签共享全局作用域，`const`/`let` 不能跨标签重复声明。每个模块应确保常量只声明一次。

**Playwright 自动化验证结论**（22:55）：
- onboarding overlay 是 z-index 400，会遮挡 nav (z-index 100)——但不是导航不响应的根本原因
- 根本原因是 `STAGE_EXP` 重复声明导致 `pet-renderer.js` 整个文件不执行，`page:enter` 的宠物渲染监听器没有注册
- 修复后全流程测试通过：onboarding 完成后宠物页/任务页/商店页全部正常渲染，零 JS 错误
- 测试方法：Playwright + npm install playwright + npx playwright install chromium
